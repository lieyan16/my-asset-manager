## 📌 项目背景

市面上的资产记账 App 大多只记录“我有什么”，却很少告诉你 **每件物品每天的真实使用成本**。

比如：买了一台 5000 元的手机，用了 1000 天 → 每天成本 = **5 元**。  
这个系统就是为了解决这个问题而设计的 —— 让你清楚知道每件资产是“真值”还是“亏了”。

同时支持 **自定义分类、位置、标签**，并提供 **局域网自动备份/同步** 方案（无需公网服务器）。

---

## ✨ 核心功能

- ✅ **资产全记录**：手机、相机、电脑、硬盘……任意物品，价格、购买日期、分类、位置、标签一目了然。
- 📊 **智能成本计算**  
  - 单件物品：`(购买价格 / 已使用天数)` 元/天  
  - 总览看板：每日总成本、最贵物品、最长使用物品、日均成本最高物品……
- 🏷️ **灵活分类体系**  
  - 自定义分类（如：数码、家居、户外）  
  - 自定义位置（家里 / 公司 / 身上）  
  - 自定义 Tag 标签（便于后期过滤检索）
- 🌓 **深色/浅色主题**（预留，可快速扩展）
- 🔄 **局域网备份与同步**（设计完成，待实现）  
  - 手机端与电脑端在同一个 WiFi 下自动发现  
  - 以 SQLite 数据库文件整体同步  
  - 冲突时由用户选择保留最新版本
- 📤 **本地数据导入/导出**（支持备份恢复）

## 🛠️ 技术栈

| 部分 | 技术 | 说明 |
|------|------|------|
| 前端（App + Web） | **Flutter** (Dart) | 一套代码构建 Android App 和 Web 端 |
| 后端（同步用） | Go / Python (FastAPI) | 轻量级 HTTP 文件服务，可选 |
| 本地数据库 | **SQLite** | 单文件，无需数据库服务，方便同步 |
| 状态管理 | Provider / GetX | 按需选择 |
| 局域网同步 | 自定义 HTTP API + 文件传输 | 设计文档已完成 |

> 💡 **特别说明**：整个系统 80% 以上的代码由 **ChatGPT + DeepSeek** 辅助生成，是 AI 辅助开发的实践项目。
